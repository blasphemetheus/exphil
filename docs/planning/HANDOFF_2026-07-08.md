## Protocol lessons (cost a failed run each — do not relearn)

- Rollout relabeling: targets = expert corrections, but the prev-action
  channel must carry the policy's ACTUAL press (`:prev_controller` frame
  override) — else "(state, own-mistake) → fix" pairs don't exist.
- Recovery taps alternate against the previously-landed input, never
  action_frame parity (af freezes on repeated frames; embedding saturates at
  af/60 cap 2.0 → coin-flip labels → plateau then NaN).
- Buttons edge, not hold: 582 held-X frames = zero jumps. Decode-side
  complement: `--press-threshold/--release-threshold` hysteresis (built,
  positive under interaction; untested solo).
- Expert safety rules must OVERRIDE the table for position-dependent hazards
  (table keys exclude x → fade-back drift walked off the edge).
- L in the air outside aerial-attack states (65-70) is an AIRDODGE.
- Drill trainer: cosine decay (warmup 1 epoch, floor at ~200 epochs of
  steps) killed the recurring late-NaN; best-epoch export ships a usable
  policy anyway if it still diverges.
- NEVER run `mix` while a training beam is live (SIGSEGV via second EXLA
  client, even with nothing to recompile). Inference-only sessions coexist.
  Automated loops run mix continuously — treat the whole loop as no-mix.
- dagger_loop.sh auto-seeds from today's >500KB replays — pass ROLLOUTS
  explicitly on mixed days or it ingests other drills'/human games.
