## OPERATIONAL RULES (new since last handoff — GOTCHAS entries owed, #8)

1. **Shared-NIF fine-lock trap**: two consumers (exphil/edifice) with
   different `fine` versions in their locks cannot share one local-exla
   NIF — the .so exports won't match the newer consumer's generated
   stubs (`Function not found ... write_to_pointer/3`). Keep dep locks
   aligned across local-nx consumers, or rebuild the NIF after lock
   bumps with NOTHING running.
2. **Loops must launch under devenv** — a bare-shell launch dies ~2 min
   in on cargo/make :enoent, buried mid-log ("produced no checkpoint",
   burned two launches). newera7+ scripts guard for make/cargo at t=0.
3. **Never XLA_TARGET=cpu on this machine** — CUDA objects link against
   the CPU xla_extension → unloadable NIF (undefined symbol:
   cudaGetErrorString). devenv default (cuda12) only.
4. **Probe Dolphin login failure**: user's Slippi launcher open during
   probe start can leave the probe's netplay-build Dolphin stuck at a
   login screen (r8_p1 lost this way). Keep the launcher closed during
   probe windows until #5 makes probes headless.
5. **pkill self-match**: `pkill -f <pattern>` in a Bash tool call whose
   own command line contains the pattern kills the shell (exit 144).
   Kill by exact PID or `pgrep -x` name.
6. Mid-loop lib edits still forbidden (loop recompiles per stage);
   mix run vs mix test build DIFFERENT envs (:dev vs :test) — a green
   test preflight does NOT mean the :dev build exists.
