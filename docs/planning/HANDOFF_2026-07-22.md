## GOTCHA CANDIDATES (write into GOTCHAS.md next touch)

- **Training-run monitoring must be liveness+heartbeat, not error-grep.**
  Check: pid alive AND log mtime advancing within N min. r16 died at
  epoch 10, undetected 7h, because the Monitor grepped for FATAL/
  diverged/NaN — and the actual death line, `** (ArgumentError)`,
  matched none of them. Grep-based monitors miss BOTH silent kills
  (SIGKILL writes nothing) AND crashes whose message you didn't
  predict; liveness+heartbeat catches every death class. Corollary:
  before theorizing about a death, READ THE LOG TAIL — the r16 stack
  trace sat in the drill log for 7h while an OOM theory was built on
  "the log just stops."
- **Launcher must log the trainer's exit status** (now does): 137 =
  SIGKILL/OOM-killer, 139 = SIGSEGV/NIF-relink, else = crash with a
  stack trace in the log. r16's "no checkpoint" message hid exit 1 —
  one echo would have prevented the misdiagnosis. A real OOM kill DOES
  log to `journalctl -k` on this box (proof: gjs killed at 54 GB RSS,
  Jul 21 13:52).
- **scenario_suite .slp don't finalize** — it stops games mid-play, so
  Slippi never writes the raw-data length; the .slp fail Peppi parse
  even on clean completion. #38 seeds need play-to-game-end.
- **Mainline Slippi needs a MAINLINE-format home** (netplay-beta layout,
  no `ishiiruka` marker). Feeding the Ishiiruka home crashes at
  online-session start (CoreTiming ScheduleEvent nullptr).
- **pgrep -f "<pat>" self-matches the running shell** when the pattern is
  in your own argv — killed 2 launches; also makes "drill running"
  false-positive. Verify with `ps -eo pid,comm`; never combine a
  pgrep-kill with the thing you're launching.
- **Axon.build seeds param init from system_time()** — pass :seed for
  reproducible fits (bit edifice SAETrainer).
- **Microbench fusion != full-graph fusion** — two block benches
  misattributed the r15 slowdown; only the real-graph profiler ranked it.
